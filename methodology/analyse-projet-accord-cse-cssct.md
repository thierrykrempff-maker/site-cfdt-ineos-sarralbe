# METHODE CFDT NEXUS
## Analyse d'un projet d'accord et preparation de l'action CSE/CSSCT

Version : 1.0

## Objectif

Cette methode formalise le raisonnement attendu du Document Intelligence Center lorsqu'un elu CFDT analyse un projet d'accord d'entreprise ou un document presente par la direction.

Le but n'est pas de produire un simple resume. Le DIC doit aider a comprendre le projet, verifier les points sensibles, preparer des questions CSE/CSSCT et transformer l'analyse en actions syndicales concretes.

## Principes obligatoires

- L'IA assiste, l'elu decide.
- Ne jamais inventer une regle juridique.
- Ne jamais inventer une jurisprudence.
- Distinguer les faits, les hypotheses et les recommandations.
- Toute hypothese sur la strategie de la direction doit etre presentee comme une hypothese a verifier.
- Une mesure legale n'est pas automatiquement une mesure socialement equilibree.
- L'avis des salaries ne doit jamais etre suppose.
- Les documents confidentiels ne doivent jamais etre ajoutes au depot public.
- Ne jamais exposer la BDESE ou des donnees personnelles.
- Les questions produites doivent etre concretes, documentees et difficiles a contourner.
- L'objectif final est toujours de transformer l'analyse en action concrete.

## Etape 1 - Comprendre pourquoi le changement est propose

Question principale : pourquoi ce changement ?

L'analyse doit rechercher :

- le probleme declare par la direction ;
- le contexte du projet ;
- la raison du calendrier ;
- les objectifs annonces ;
- les elements factuels avances ;
- les indicateurs utilises par la direction ;
- les solutions alternatives eventuellement etudiees.

Questions de relance :

- Quel probleme concret ce changement cherche-t-il a resoudre ?
- Depuis quand ce probleme est-il identifie ?
- Sur quels indicateurs repose ce constat ?
- Quelles donnees objectives pouvez-vous communiquer au CSE ?
- Quelles autres solutions ont ete etudiees ?
- Pourquoi ont-elles ete ecartees ?
- Pourquoi ce projet est-il presente maintenant ?

Sortie attendue : comprendre la justification officielle et le contexte reel du projet.

## Etape 2 - Verifier la conformite juridique

Le systeme doit rechercher les regles potentiellement applicables dans l'ordre pertinent selon le sujet :

- accords d'entreprise applicables ;
- convention collective applicable, notamment Convention collective nationale des industries chimiques ;
- Code du travail ;
- obligations d'information et de consultation du CSE ;
- regles de sante, securite et conditions de travail ;
- reglement interieur si pertinent ;
- jurisprudence recente et pertinente.

L'analyse ne doit jamais se limiter a conforme / non conforme.

Elle doit distinguer :

- conforme en apparence ;
- point a verifier ;
- risque juridique identifie ;
- information manquante ;
- interpretation possible ;
- procedure obligatoire ;
- delai a surveiller.

Sortie attendue : une lecture exploitable par un elu, avec les documents a demander avant de poursuivre.

## Etape 3 - Analyser a qui profite le changement

Question centrale : a qui profite reellement ce changement ?

Benefices possibles pour l'entreprise :

- flexibilite ;
- productivite ;
- reduction des couts ;
- simplification organisationnelle ;
- disponibilite accrue ;
- reduction des contraintes ;
- optimisation des effectifs ;
- transfert de responsabilites.

Benefices possibles pour les salaries :

- remuneration ;
- repos ;
- previsibilite ;
- securite ;
- conditions de travail ;
- equilibre vie professionnelle / vie personnelle ;
- evolution professionnelle ;
- garanties supplementaires.

Contraintes a identifier :

- qui supporte la charge supplementaire ;
- qui perd du repos ;
- qui assume la flexibilite ;
- qui supporte le risque ;
- qui supporte le cout ;
- qui subit de nouvelles obligations.

Regle methodologique : une mesure peut etre juridiquement possible mais socialement desequilibree. CFDT Nexus doit analyser separement la legalite et l'equilibre social.

## Etape 4 - Analyser les consequences concretes pour les salaries

L'analyse doit rechercher les consequences sur :

- charge de travail ;
- horaires ;
- temps de travail ;
- temps de repos ;
- pauses ;
- effectifs ;
- organisation des equipes ;
- fatigue ;
- sante ;
- securite ;
- risques professionnels ;
- risques psychosociaux ;
- remuneration ;
- primes ;
- majorations ;
- astreintes ;
- conges ;
- organisation familiale ;
- egalite de traitement ;
- categories particulierement exposees ;
- consequences a court terme ;
- consequences a moyen terme ;
- consequences indirectes possibles.

Le systeme doit transformer chaque risque identifie en question exploitable.

Exemples preferes :

- Quelle evaluation de l'impact sur la charge de travail a ete realisee ?
- Sur quelles donnees cette evaluation repose-t-elle ?
- Quels metiers, postes et equipes ont ete etudies ?
- Quels risques supplementaires ont ete identifies ?
- Quelles mesures de prevention sont prevues ?
- Quels indicateurs permettront au CSE de mesurer les effets reels apres 3 mois et 6 mois ?

## Etape 5 - Rechercher la jurisprudence utile et construire l'argumentation

Le systeme doit prevoir une etape de recherche jurisprudentielle ciblee.

Il ne doit jamais produire une simple liste d'arrets.

Pour chaque jurisprudence utilisee, l'analyse future devra examiner :

- la juridiction ;
- la date ;
- le numero de decision ;
- les faits de l'affaire ;
- la question juridique ;
- la solution retenue ;
- la proximite avec le cas analyse ;
- les differences avec le cas analyse ;
- l'utilite pour l'argumentation ;
- les limites de son utilisation.

Regle : une jurisprudence ne doit jamais etre utilisee uniquement parce que son titre ou son resume contient des mots similaires au dossier.

Rubriques a produire :

- jurisprudences potentiellement pertinentes ;
- pourquoi cette decision peut etre utile ;
- differences avec notre situation ;
- argument possible ;
- limites et points a verifier.

## Etape 6 - Preparer un resume pour les salaries concernes

CFDT Nexus doit transformer l'analyse technique en information accessible.

Le resume salarie doit expliquer simplement :

- la situation actuelle ;
- ce que la direction propose ;
- ce qui changerait ;
- les salaries concernes ;
- les avantages eventuels ;
- les risques identifies ;
- les consequences concretes possibles ;
- les points encore flous ;
- les questions posees par la CFDT.

Le ton doit etre clair, factuel, accessible, non alarmiste et sans jargon juridique inutile.

Objectif : informer avant de demander un avis.

## Etape 7 - Recueillir l'avis des salaries

Prevoir une sortie permettant de preparer une consultation :

- questionnaire court ;
- sondage ;
- questions ouvertes ;
- points a discuter en reunion ;
- categories de salaries a consulter en priorite.

Le systeme ne doit jamais supposer l'avis des salaries.

Il doit distinguer :

- analyse technique ;
- position CFDT ;
- avis reel des salaries concernes.

## Etape 8 - Construire une alternative plus favorable

Si le projet est defavorable ou rejete par les salaries concernes, CFDT Nexus doit aider a construire une alternative.

Methode :

1. Identifier le probleme initial que la direction cherche a resoudre.
2. Identifier les contraintes reelles de l'entreprise.
3. Identifier les consequences negatives de la solution proposee.
4. Identifier les priorites exprimees par les salaries.
5. Construire une solution alternative.
6. Verifier sa faisabilite juridique.
7. Identifier les avantages pour les salaries.
8. Identifier les avantages possibles pour l'entreprise.
9. Preparer les arguments de negociation.
10. Preparer les reponses aux objections previsibles de la direction.

Formulation cible :

"Nous comprenons le probleme que vous cherchez a resoudre. Nous contestons la solution proposee pour les raisons suivantes. Voici une alternative qui permettrait de repondre au probleme tout en protegeant mieux les salaries."

## Sorties CSE

Les questions CSE doivent porter notamment sur :

- justification du projet ;
- donnees economiques et organisationnelles ;
- effectifs ;
- organisation ;
- calendrier ;
- consequences sociales ;
- criteres de decision ;
- alternatives etudiees ;
- moyens ;
- indicateurs de suivi ;
- engagements de la direction.

## Sorties CSSCT

Les questions CSSCT doivent porter notamment sur :

- sante ;
- securite ;
- charge de travail ;
- fatigue ;
- horaires ;
- repos ;
- risques professionnels ;
- risques psychosociaux ;
- evaluation des risques ;
- mesures de prevention ;
- DUERP ;
- plan d'action ;
- indicateurs de suivi ;
- retour d'experience ;
- populations particulierement exposees.

## Synthese pour l'elu

La synthese finale doit afficher :

- principaux changements ;
- principaux risques ;
- points juridiquement sensibles ;
- informations manquantes ;
- questions prioritaires ;
- documents a demander ;
- arguments disponibles ;
- avis des salaries a recueillir ;
- alternatives a travailler ;
- prochaines actions recommandees.

## Scenario fictif de demonstration

Scenario : projet fictif de modification de l'organisation du travail en equipes postees.

Hypothese de depart :

- la direction propose de modifier les horaires d'equipes ;
- le delai de prevenance serait reduit ;
- certaines equipes auraient davantage de flexibilite imposee ;
- l'objectif annonce serait une meilleure adaptation a la charge de production ;
- aucun accord INEOS reel ni document confidentiel n'est utilise.

Demonstration attendue :

- comparer le projet avec l'existant ;
- demander le motif du changement et les indicateurs utilises ;
- verifier la conformite avec les accords applicables, la convention collective et le Code du travail ;
- analyser l'equilibre entre benefices entreprise et impacts salaries ;
- evaluer les consequences concretes sur charge, fatigue, repos, remuneration et vie personnelle ;
- produire des questions CSE ;
- produire des questions CSSCT ;
- preparer une consultation des salaries ;
- proposer une alternative plus favorable, par exemple une phase test, un delai de prevenance plus protecteur, des compensations et un suivi CSE/CSSCT.
