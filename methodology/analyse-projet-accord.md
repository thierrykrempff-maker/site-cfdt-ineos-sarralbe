# Analyse d'un projet d'accord

Statut : méthode de référence V1

Usage : Document Intelligence Center - profil "Projet d'accord"

## Objectif

Cette méthode sert à analyser un projet d'accord, un avenant ou une proposition de texte soumis à la CFDT.

Elle ne vise pas seulement à résumer le document. Elle doit aider le représentant du personnel à comprendre :

- ce que le texte change réellement ;
- qui est concerné ;
- quels sont les bénéfices possibles ;
- quels sont les risques ou angles morts ;
- quelles questions poser ;
- quels documents demander ;
- quelle stratégie préparer avant échange, réunion ou négociation.

## Principe construit avec le Product Owner

L'analyse doit suivre une logique syndicale simple :

1. Comprendre le document.
2. Identifier ce qui change pour les salariés.
3. Repérer les points favorables, défavorables ou ambigus.
4. Vérifier les textes et accords concernés.
5. Préparer les questions utiles.
6. Proposer des pistes de stratégie.
7. Laisser la décision finale à l'humain.

Le Document Intelligence Center devra donc produire une aide à la décision, jamais une décision automatique.

## Quand utiliser cette méthode

Utiliser cette méthode pour :

- un projet d'accord d'entreprise ;
- un avenant à un accord existant ;
- un projet de protocole ou charte ayant des effets collectifs ;
- une note de cadrage préparatoire à une négociation ;
- une version comparative d'un accord.

Ne pas l'utiliser seule pour :

- une situation individuelle disciplinaire ;
- une fiche de paie ;
- une convocation ;
- un PV CSE ;
- une décision de justice.

Ces cas feront l'objet de méthodes dédiées.

## Informations nécessaires

Avant d'analyser, vérifier si les éléments suivants sont disponibles :

- titre du document ;
- date du projet ;
- version du projet ;
- auteur ou origine du texte ;
- périmètre concerné ;
- salariés ou catégories concernés ;
- accord existant remplacé ou modifié ;
- délai de réponse ou calendrier de négociation ;
- documents annexes cités ;
- contexte connu : NAO, réorganisation, temps de travail, rémunération, QVCT, santé sécurité, emploi.

Si une information manque, l'analyse doit le signaler clairement.

## Étapes d'analyse

### 1. Qualification du document

Identifier la nature du texte :

- accord nouveau ;
- avenant ;
- projet de négociation ;
- version de travail ;
- document préparatoire ;
- texte déjà signé.

Préciser le niveau de confidentialité : Public, Interne ou Confidentiel.

### 2. Synthèse courte

Produire une synthèse en langage clair :

- sujet principal ;
- objectif affiché ;
- changements clés ;
- populations concernées ;
- échéance importante ;
- prochaine action recommandée.

La synthèse doit être courte et directement exploitable.

### 3. Ce qui change

Comparer le projet avec la situation existante lorsque c'est possible.

Classer les changements par thème :

- rémunération ;
- temps de travail ;
- congés ;
- organisation ;
- emploi ;
- formation ;
- santé, sécurité et conditions de travail ;
- égalité professionnelle ;
- droits syndicaux ;
- procédure ou gouvernance.

Pour chaque changement, distinguer :

- fait écrit dans le projet ;
- interprétation possible ;
- point à vérifier ;
- conséquence probable pour les salariés.

### 4. Impact salariés

Identifier concrètement :

- qui gagne quelque chose ;
- qui peut perdre quelque chose ;
- qui est exclu du dispositif ;
- quelles situations particulières sont oubliées ;
- quels effets peuvent apparaître à court, moyen ou long terme.

L'analyse doit rester centrée sur les conséquences pratiques pour les salariés.

### 5. Points forts

Lister les éléments favorables :

- avancées sociales ;
- sécurisation d'un droit ;
- clarification utile ;
- amélioration d'un dispositif existant ;
- engagement mesurable de l'employeur ;
- délai ou suivi prévu.

Chaque point fort doit être relié à une clause ou à une conséquence concrète.

### 6. Points faibles et risques

Repérer :

- formulations floues ;
- droits conditionnés ;
- absence de garanties ;
- renvois non maîtrisés ;
- exceptions trop larges ;
- effets différenciés entre salariés ;
- risques de perte d'acquis ;
- indicateurs absents ;
- clause de révision, dénonciation ou durée problématique.

Un risque doit être formulé sobrement : ce qui pourrait arriver, pourquoi, et ce qu'il faut vérifier.

### 7. Textes et accords concernés

Identifier les références à vérifier :

- Code du travail ;
- convention collective applicable ;
- accords d'entreprise existants ;
- règlement intérieur si concerné ;
- jurisprudence utile si le sujet est sensible ;
- usages ou pratiques internes connus.

L'analyse ne doit pas inventer une base juridique. Si la référence n'est pas disponible, elle doit indiquer : "référence à vérifier".

### 8. Questions à poser

Préparer des questions utiles pour la réunion ou la négociation.

Les questions doivent couvrir :

- intention réelle de l'employeur ;
- salariés concernés ;
- cas particuliers ;
- garanties ;
- durée ;
- suivi ;
- indicateurs ;
- moyens ;
- articulation avec les accords existants ;
- conséquences en cas de désaccord ou de non-application.

Chaque question doit avoir un objectif clair.

### 9. Documents à demander

Identifier les documents nécessaires :

- version précédente ou accord remplacé ;
- tableau comparatif ;
- données chiffrées ;
- liste des populations concernées ;
- calendrier ;
- étude d'impact ;
- annexes techniques ;
- accords liés ;
- compte rendu de réunion précédente.

Ne demander que les documents utiles à la compréhension ou à la négociation.

### 10. Stratégie syndicale

Préparer des pistes de stratégie, sans décider à la place de l'humain :

- arguments favorables ;
- arguments de vigilance ;
- points non négociables possibles ;
- concessions envisageables ;
- amendements à proposer ;
- demandes de clarification ;
- conditions de signature ;
- motifs possibles de refus ou d'abstention.

La stratégie doit être présentée comme une aide, pas comme une instruction.

## Livrable attendu du Document Intelligence Center

Pour un projet d'accord, le moteur devra produire au minimum :

1. Synthèse courte.
2. Ce qui change.
3. Salariés concernés.
4. Points forts.
5. Points faibles.
6. Risques ou angles morts.
7. Textes et accords à vérifier.
8. Questions à poser.
9. Documents à demander.
10. Délais à surveiller.
11. Pistes de stratégie.
12. Prochaine action recommandée.

## Grille de lecture rapide

| Axe | Question centrale | Sortie attendue |
| --- | --- | --- |
| Compréhension | De quoi parle le texte ? | Synthèse claire |
| Changement | Qu'est-ce qui change ? | Liste des modifications |
| Impact | Qui est concerné et comment ? | Effets salariés |
| Risque | Qu'est-ce qui peut poser problème ? | Points faibles et vigilances |
| Référence | Quels textes vérifier ? | Références ou points à confirmer |
| Action | Que faire ensuite ? | Questions, documents, stratégie |

## Limites

Cette méthode ne remplace pas :

- la lecture humaine du projet ;
- l'avis d'un juriste ;
- la validation d'une organisation syndicale ;
- la négociation ;
- la décision de signer ou non.

Toute conclusion sensible doit être relue et validée par un représentant compétent.

## Principe de sortie

Une analyse réussie doit permettre au représentant CFDT de répondre à quatre questions :

1. Qu'est-ce que ce projet change ?
2. Quels salariés sont concernés ?
3. Quels points doivent être sécurisés ?
4. Que dois-je faire ensuite ?
