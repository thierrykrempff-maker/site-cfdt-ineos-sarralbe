## Reference principale

Le document [MANIFESTE_CFDT_NEXUS.md](docs/MANIFESTE_CFDT_NEXUS.md) est la reference principale du projet.

Avant toute evolution importante :

- verifier que la demande respecte le manifeste ;
- si une demande est contraire au manifeste, expliquer pourquoi ;
- proposer une alternative conforme ;
- citer le principe du manifeste concerne lorsque c'est pertinent.

Les decisions d'architecture et de developpement doivent rester coherentes avec les principes du manifeste :

- etre proche des salaries ;
- expliquer clairement ;
- defendre sans diviser ;
- agir avec serieux, transparence et determination ;
- informer, accompagner, alerter si necessaire et proposer des solutions concretes.
